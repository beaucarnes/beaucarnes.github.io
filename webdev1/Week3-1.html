##CSS3 Features
###CSS3 Vendor Prefix Not all browsers support all of the newer CSS3 properties, which means we sometimes need to create rules for certain browsers

border-radius usage

                            
                .example {
                  -moz-border-radius: 20px;
                  -ms-border-radius: 5%;
                  -webkit-border-radius: 1.2em;
                  -o-border-radius: 10px;
                  border-radius: 10%;
                }
###Vendor prefixes Order matters! The non-prefixed property should always go last.
###Handy CSS3 properties There are a great many very handy CSS3 properties that you may need to use on your current and future projects. They can add a level of polish to your project that really sells it. That said, we dont have a time to go over more than a few of them in detail. Here is a list that you can investigate further.
###CSS3 properties Property | Description --- | --- `border-radius` | creates rounded corners or circular borders for elements `rgba` | An alternate way to define color and alpha `opacity` | Controls the transparency of an element `box-shadow` | Adds a calculated shadow effect to an element `text-shadow` | Adds a calculated shadow effect to an element `transforms` | Adds the ability to transform HTML elements (rotation, scale, translate) `animations` | Allows for one more or property changes to be applied as an animation
## Responsive Design
###Responsive Design Changes your website presentation in relation to the screen size
###Media Queries This is done via Media Queries. Media queries are able to detect a bunch of things about the device with which you're viewing a site. The browser can use specifically developed CSS styles for those different screen sizes.

Media Query example

                            
              @media (max-width: 600px) {
                /* specific styles for this device size */
                body{
                  font-size: 1.1em;
                }
              }
###Things to do with media queries - resize images / font - collapse navigation - remove some larger content (video / images)

##Domains
###What are domains? Domains (e.g., www. facebook.com) are registered with the Domain Name System through various providers The Domain Name System allows a website’s IP address (192.0.43.10) to be mapped to a specific domain name (www.example.com)
###Dedicated Domain Names www.grandcircus.co vs. www.grandcircusco.tumblr.com
###Domain Names 101 1. As short as possible 1. Easy to type 1. Easy to remember 1. Unique 1. Intuitive 1. Avoid hyphens & numbers 1. End in “.com” if possible
###Domain Name Vendors Namecheap (www.namecheap.com) Hover (www.hover.com) DreamHost (www.dreamhost.com)
##Servers/Hosting
###What does it mean? Websites are hosted on web servers, which are large computers connected to a network Web servers “serve up” content and services, which can include web sites and applications, games, etc.
###How it works 1. Type www.google.com into the address bar 1. DNS connects the browser (user agent) to Google’s hosting servers 1. Google’s servers send back content for your browser to display on your computer (client) (Sometimes, code must be compiled by the server before it can be sent back to your machine.)
##SERVERS & HOSTING – WHO READS WHAT?
###Client-side Client-side programming languages can be parsed on the client side within your browser after a web page is loaded. ###Server-side Server-side programming languages are parsed by the server and run scripts before the page is loaded, not after.
###Client-side Languages - Markup languages (HTML & CSS) - JavaScript
###Server-side Languages - Ruby - Python - PHP - Perl - Java - C#.NET
###Hosting Vendors HostGator (www.hostgator.com) DreamHost (www.dreamhost.com) BlueHost (www.bluehost.com)
